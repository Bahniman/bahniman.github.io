The Trust Layer for the Agent Economy

Five working prototypes. Live demos in your browser, no installation. By Bahniman Talukdar.

The idea in plain English. AI assistants ("agents") are starting to do real things for people: spending money, booking services, checking work, making records. That creates a new problem nobody has solved: how do we trust what software does in the real world? Each prototype below answers one piece of that question, with working code you can try right now.

The five prototypes

Surety

Plain English: a permission slip for your AI assistant. You sign what it may spend and where; anything else gets blocked or waits for your approval — with a tamper-proof record of everything.

Try the live demo Code on GitHub

GroundTruth

Plain English: when real-world work (like a road being built) is verified with photos + AI + an accountable engineer's signature, the payment can happen in days instead of months. Watch a measurement become money.

Try the live demo Code on GitHub

Heirloom

Plain English: a company's memory (decisions, reasons, meeting notes) stored in one safe place the company owns — so it survives when employees leave or AI vendors change. Ask the same question as three different roles and see the difference.

Try the live demo Code on GitHub

Turnstile

Plain English: a growing share of online shoppers are not people — they are AI assistants shopping on someone's behalf. Turnstile tells a store how many, from where, and helps the store sell to them properly.

Try the live demo Code on GitHub

Windtunnel

Plain English: pilots rehearse landings in simulators; companies change prices with a spreadsheet and hope. Windtunnel rehearses a price change against thousands of simulated customers first. Move the slider, see the outcome.

Try the live demo Code on GitHub

Why these five together?

Agents acting in the world create a trust vacuum: authority (Surety), physical reality (GroundTruth), memory (Heirloom), commerce (Turnstile), and consequences (Windtunnel). One thesis, five working answers.
