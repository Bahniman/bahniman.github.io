Turnstile

Your store's analytics assume visitors are human. A growing share aren't.

The business problem, precisely. AI assistants now browse, compare and buy on their owners' behalf — and to an e-commerce stack they're invisible or, worse, misread. Three expensive failures are already happening: (1) lost sales — fraud filters see "no mouse movement, instant checkout" and decline legitimate delegated purchases; (2) wasted spend — retargeting ads are served to software that will never see them; (3) broken data — agent sessions pollute funnels and A/B tests, so the store optimizes against noise. Turnstile is the missing instrument: it classifies every session (human / agent / which platform), recovers the falsely-declined orders, attributes agent revenue correctly, and then serves detected agents a machine-readable storefront that converts better than a webpage.

Then stop showing agents a webpage

What an agent fights through today:
🍪 cookie banner → 🎠 hero carousel → 📧 newsletter pop-up → 💬 chat widget → price hidden behind a size selector → checkout in 6 steps

Result: mis-parsed prices, abandoned carts, wrong variants ordered.
What Turnstile serves a detected agent instead:
GET /.well-known/agent-offers.json

{ "sku": "SNK-DUNE-42",
  "name": "Dune Runner sneaker, UK 8",
  "price": 4299, "in_stock": true,
  "ships_in_days": 2, "returns": "14-day",
  "buy": "/api/checkout?sku=SNK-DUNE-42" }
Clean data means fewer wrong orders for the agent and a correctly-attributed sale for the merchant. Being detected becomes a benefit — which is what turns an adversarial arms race into a standard both sides adopt.

Why merchants pay for this

🛑Recovered declines. Every legitimate agent order the fraud filter kills is pure lost margin. Turnstile whitelists verified delegated purchases.
🎯Unpolluted data. Funnels, A/B tests and ad audiences with the software filtered out — the store optimizes against humans again.
🛍️A new channel. Agent-attributed revenue becomes a line the merchant can grow — with the offer feed as its storefront.
Revenue model: the free snippet answers one question ("what % of your traffic is agents?") and spreads itself — every merchant wants their number, and a quarterly published benchmark makes it a category. Paid tiers: platform attribution + fraud-whitelist + offer feed; long-term, basis points on feed-attributed revenue. The moat is the labeled traffic corpus — detection improves with every store on the network, and a latecomer can't download that.
Jargon decoder
TermMeaning
Agent trafficStore visits made by AI software acting for a person — browsing, comparing, buying.
Classifier / signalsThe logic that decides human-or-agent from behavioral clues: mouse movement, load patterns, timing, whether the page's script ever ran.
False declineA legitimate order rejected by a fraud filter because the buyer "behaved like a bot" — which a delegated agent does.
AttributionCrediting a sale to its true source — e.g. "via OpenClaw agent, delegated by its owner."
Offer feed / AEOA machine-readable storefront (price, stock, terms, checkout endpoint) at a fixed address — SEO's playbook, replayed for agents.
